## 命名空间

命名空间是一种将脚本分组的方法，避免类和方法名冲突。当多个脚本存在相同的名称时，命名空间是一种很好的解决方案。

下面是一个简单的命名空间示例：

```csharp
// GameManager.cs
namespace MyGame
{
    public class GameManager
    {
        // 游戏初始化
        public void Init()
        {
            // ...
        }
    }
}

// InputController.cs
namespace MyGame.Input
{
    public class InputController
    {
        // 处理用户输入
        public void HandleInput()
        {
            // ...
        }
    }
}

// PlayerController.cs
namespace MyGame.Characters
{
    public class PlayerController
    {
        // 处理玩家移动
        public void Move()
        {
            // ...
        }
    }
}

// EnemyController.cs
namespace MyGame.Characters
{
    public class EnemyController
    {
        // 处理敌人移动
        public void Move()
        {
            // ...
        }
    }
}
```

在这个示例中，我们定义了一个游戏管理器，负责游戏的初始化。在命名空间“Input”下，我们定义了一个输入控制器，负责处理用户输入。在命名空间“Characters”下，我们定义了一个玩家控制器和一个敌人控制器，负责处理角色的移动。

这是一个很好的组织方式，可以帮助保持代码的清晰度，并方便以后的维护和扩展。
